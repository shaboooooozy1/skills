# Email Command Center

## When to use this skill
Use this skill when the user asks to:
- Triage or prioritize inbox messages
- Draft new emails or reply to existing threads
- Rewrite emails for tone, clarity, or brevity
- Summarize long threads and pull decisions/action items
- Generate follow-ups, reminders, or nudges
- Create structured outreach sequences (e.g., intro, follow-up, final check-in)

## Workflow

1. **Classify the email task**
   - Triage, draft, rewrite, summarize, follow-up, or sequence.

2. **Collect missing context (only if needed)**
   - Audience/recipient
   - Goal and desired outcome
   - Tone (formal, friendly, direct, executive)
   - Constraints (length, deadlines, legal/sensitive boundaries)

3. **Produce the output in the right format**
   - **Triage:** Priority buckets (Urgent, Important, Low), suggested next action, and draft snippets.
   - **Draft/Reply:** Subject line + body with clear CTA and timeline.
   - **Rewrite:** 2-3 alternatives with brief tone labels.
   - **Summary:** 3-part output: Key points, Decisions, Action items (owner + due date if available).
   - **Sequence:** Numbered touchpoints with send timing and short email copy per touchpoint.

4. **Run a quality pass before finalizing**
   - Confirm asks are explicit and dates are concrete.
   - Remove ambiguity and unnecessary jargon.
   - Check tone alignment with recipient seniority and relationship.
   - Keep writing concise and scannable.

## Output standards
- Prefer short paragraphs and bullets for readability.
- Include a specific call to action in most outbound drafts.
- If dates are relative ("tomorrow", "next week"), include exact dates when possible.
- For sensitive topics, default to neutral and factual phrasing.
